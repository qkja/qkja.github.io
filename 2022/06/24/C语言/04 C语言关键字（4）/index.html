两个问题

问：为什么要定义类型？答：根据不同的类型在内存中开辟不同大小的一块空间。问：为什么要定义这么多类型？我们的应用场景是很复杂的，不同应用场景要求不同大小的空间，比如说，一个人的年龄、身高等变量可以定义成short型，定义成int型就浪费了空间。

数据在内存中的存储（重点）

我们都知道，数据在内存中是以二进制的形式存入的，如何将整型 浮点型存入内存呢？下面就是我们要研究的。

浮点型的存储

存储

根据国际标准IEEE（电气和电子工程协会） 754，任意一个二进制浮点数V可以表示成下面的形式：NUM = （-1）^S * M * 2 ^E

  • S 的取值 正数 0 负数 1
  • M 表示有效数字，大于等于1，小于2
  • E 表示指数位

float型

1个符号位 8个指数位 23个有效数字位

double 型

1个符号位 11个指数位 52个有效字符位

关于 M 的值

IEEE 754规定，在计算机内部保存M时，默认这个数的第一位总是1，因此可以被舍去，只保存后面的xxxxxx部分。比如保存1.01的时候，只保存01，等到读取的时候，再把第一位的1加上去。这样做的目的，是节省1位有效数字。以32位浮点数为例，留给M只有23位，将第一位的1舍去以后，等于可以保存24位有效数字。

关于E的值

首先，E为一个无符号整数（unsigned int） 这意味着，如果E为8位，它的取值范围为0255；如果E为11位，它的取值范围为02047。但是，我们知道，科学计数法中的E是可以出现负数的，所以IEEE 754规定，存入内存时E的真实值必须再加上一个中间数，对于8位的E，这个中间数是127；对于11位的E，这个中间数是1023。比如，2^10的E是10，所以保存成32位浮点数时，必须保存成10+127=137，即10001001。

存储样例 - 以float型为例

所以我们就得到9.5 在内存的二进码

0100 0001 0001 1000 0000 0000 0000 0000

4 1 1 8 0 0 0 0

41 18 00 00 (小端存储）

取出

下面是如何存储的规则

  • E不全为0，也不全为1

    E的真实值 E - 127 M 加上 1

  • E全为0

    E的真实值-127 NUM 无限接近于 0

  • E全为1

    E的真实值 128 NUM 无限接近于正无穷 或 负无穷

浮点数的精度损失

虽然我们有了浮点数的存储规则，但是总会有不能够用有限的二进制来表示，就像我们有无限不循环小数一样，总会出现约等于，这就是浮点数的精度问题

现象一

#include<stdio.h>
#include<windows.h>

int main()
{
  double d = 3.6;
  printf("%.50lf", d);
  return 0;
}

现象二

#include<stdio.h>
#include<windows.h>

int main()
{
  double x = 1.0;
  double y = 0.1;
  printf("%.50f\n", x - 0.9);
  printf("%.50f\n", y);
  return 0;
}

现象三

#include<stdio.h>
#include<windows.h>

int main()
{
  double x = 1.0;
  double y = 0.1;
  if ((x - 0.9) == 0.1)
  {
    printf("You can see me!!!\n");
  }
  else
  {
    printf("oops!!!\n");
  }
  return 0;
}

结论

浮点数绝对不能通过 == 进行比较 计算机很严格

浮点数如何比较

两个浮点数的比较通过他们的相减的得到的值小于某个值大于某个值，这个值就是 精度 精度可以定义 编译器自带

说人话,就是下面的的一张图

//我们认为下面的方法就可以了
if ((x - y) > - 精度 && (x - y) < 精度)
  { 
    //也可以使用fabs（x-y）    ----    fabs()函数求绝对值
  }

也就是两个数相减的结果在这个范围内就认为两数相等

样例

#include<stdio.h>
#include<windows.h>
#include<math.h>
#define EPSILON 0.000000001   //自己定义的精度

int main()
{
  double x = 1.0;
  double y = 0.1;
  /*if ((x - 0.9) == 0.1)*/
  if (fabs((x - 0.9)-0.1) <  EPSILON)
  {
    printf("You can see me!!!\n");
  }
  else
  {
    printf("oops!!!\n");
  }
  return 0;
}

系统自带精度

编译器会自带一些精度 VS中要引用<float.h>头文件

大小端问题

什么是大小端问题？先看现象：

我们预测的-5应该是以0xff ff ff fb形式存储，但事实怎们反过来了？这就是大小端问题。 int a = 0x11223344; 该变量在内存中如何存的，取时就如何取，但是有顺序的存放会更简洁，所以就保留了前两种方式，也就是大端存储和小端存储。

本质：

两种存储方案本质上都是数据和空间按照字节为单位的一种映射关系。

两者区别：

大端：低权值位放在高地址，高权值位放在低地址。 大端：低权值位放在地地址，高权值位放在高地址。

如何判断是大端存储还是小端存储

#include<stdio.h>
#include<windows.h>
#include<string.h>

int Is_sto(void)
  {
      int a = 1;
      //1   ---    00 00 00 01
      //大端 返回 0
      //小端 返回 1
      return (char)a;
}
int main()
{
  
      if (0 == Is_sto())
        printf("大端\n");
      else
        printf("小端\n");
      system("pause");
      return 0;
}

数据的存储与类型没有关系（重点）

我们先看下面一道例题，解释一下标题的的意思

#include<stdio.h>
#include<Windows.h>

int main()
{
  int a = 10;
  unsigned int b = -20;
  printf("%d\n", a + b);
  printf("%u", a + b);
  system("pause");
  return 0;
}

为什么会出现这种情况？

看看a+b在计算机中存储的数据当我们使用%d进行打印时，该数据解释为 int型使用%u打印时，解释为unsigned int 型这就是标题的意思

数据类型的比较

C语言有没有布尔类型

关于这个话题我们说详细一些，尽量把这个说的仔细

  • C语言在C99标准之前没有buerleixing
  • C99标准下增加了布尔类型
  • 大部分的编译器对C99的标准不太好

C99标准下有布尔类型 叫_Bool

为啥叫_Bool 这么难看

是因为在布尔类型出来之前，我们部分程序员自己宏定义了一些名称，这里避免和之前的代码产生冲突，造成bug

为啥叫_Bool没事，它就不会冲突吗

_Bool是保留字保留字，指在高级语言中已经定义过的字，程序员不能再将这些字作为变量名、过程名或函数名使用，保留字包括关键字和未使用的保留字。

说明_Bool、bool、BOOL的联系

_Bool

_Bool就是一种C99标准下定义的类型

  • 在不支持C99的编译器需要引用标准库**<stdbool.h>**
  • 是一个字节
#include<stdio.h>
#include<stdbool.h>

int main()
{
  _Bool fag = 1;   //可以将1赋值给他
  fag = true;
  if (fag)
  {
    printf("%d", sizeof(fag));
  }
  return 0;
}

bool

我们要记住在C++里面我们的布尔类型是bool，C/C++是相通的，避免我们在移植出现错误，我们将_Bool宏定义成了bool，本质一样

BOOL

BOOL本质上是int型，是微软自己定义的，只能在自己的产品中使用

  • TRUE　和　FALSE　都是大写的
  • 需要引用标准库**<windows.h>**
  • TRUE 就是1 FALSE就是0
#include<stdio.h>
#include<windows.h>

int main()
{
  BOOL x = TRUE;
  printf("%d", sizeof(BOOL));
  return 0;
}
